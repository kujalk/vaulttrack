Security

Military-grade encryption. Zero trust.

Your master password is never stored — not even hashed. The encryption key chain is mathematically derived at runtime and discarded when you log out.

  • 🔑
    Argon2id key derivation
    Memory-hard KDF (64 MB, 3 passes) — the same algorithm used by modern password managers. GPU brute-force is computationally infeasible.
  • 🔒
    AES-256-GCM authenticated encryption
    Every database record is individually encrypted. Tampering is detected. Random nonce on every write.
  • 🧱
    MEK / KEK split key pattern
    Changing your password re-wraps the key — no bulk re-encryption needed. Zero downtime, zero data loss.
  • 🖥️
    Local only
    Data lives in %APPDATA%\VaultTrack on Windows and ~/Library/Application Support/VaultTrack on macOS. It never leaves your machine.
// Key chain (runtime only — nothing stored in plain text) password ──Argon2id──► KEK (in RAM) │ AES-256-GCM ▼ MEK (Master Key, in RAM) │ AES-256-GCM × N ▼ investments.db ← only ciphertext on disk // What's stored in auth.dat: { "salt": base64(32 random bytes), "encrypted_mek": AES-GCM(KEK, MEK) } // Your password → nowhere.

FAQ

Common questions

What happens if I forget my password?
Your data cannot be recovered. The master password is never stored anywhere — not even hashed. This is by design: it means no one (including us) can access your data without the password. Use a password manager to store it safely.
Can I use this on multiple devices?
Yes. Copy your auth.dat and investments.db files (or use the Excel export) to the new device. The app works fully offline on each machine with the same password.
Is the database file safe to back up to cloud storage?
Yes. The database contains only AES-256-GCM ciphertext — no plaintext data. It is safe to back up to Dropbox, OneDrive, or any cloud storage without exposing your financial data.
How does currency conversion work?
The app uses open.er-api.com (free, no key required) by default. You can also enter a paid API key for higher rate limits, or enter manual exchange rates that override API rates.
Can someone extract my data from the .exe file?
No. The binary contains no user data. Even with full decompilation, an attacker still needs your investments.db and your password. Without the password, breaking Argon2id + AES-256-GCM is computationally infeasible.
What's the Excel backup for?
The Excel export creates a human-readable backup of all your records, templates, and exchange rates. You can open it in Excel or LibreOffice, review it, and re-import it. Existing records are never duplicated on import.
